# 截图 → 跨平台文档 1:1 还原

## 何时用

从一个文档平台（钉钉/飞书/Notion/Confluence/语雀/印象笔记...）截了一张或多张图，想在另一个平台**逐字逐格**复刻一份。

## 输入

| 必填 | 内容 | 默认 |
|---|---|---|
| ✅ | 一张或多张截图（按出现顺序提供）| - |
| ✅ | **目标平台** | 飞书文档 |
| ◯ | **源平台** | 自动识别 |

调用方式（任选）：

- `按截图生成飞书可粘贴的1:1还原内容`（默认目标=飞书）
- `源=钉钉文档，目标=飞书文档，按截图还原`
- `把这些 Notion 截图整成语雀可粘贴版本`

## 输出格式（AI 标准回复结构）

````markdown
[如果有标题，给 markdown 标题]

```text 或 markdown
<可直接粘贴的内容>
```

## 关键还原点

| 截图特征 | 我的处理 |
|---|---|

## 粘贴方式

1. ...
2. <如有平台特定坑，写在这里>
````

ASCII art / 树形图额外给：基线列 X、各行 padding 数。

## 1 · 内容类型识别清单

| 视觉特征 | 结构块 | 输出格式 |
|---|---|---|
| 字体显著大于正文、加粗、无前缀符号 | 标题 | `#` `##` `###`（按相对字号定级）|
| 普通正文 | 段落 | 直接文本 |
| `• ` `· ` `- ` 前缀 | 无序列表 | `-` |
| `1. 2.` 或 ① ② ③ | 有序列表 | `1.` |
| 矩形网格 + 表头加粗 | 表格 | markdown table |
| 等宽字体 + 灰底/深底框 | 代码块 | ` ``` ` 包装 |
| 代码块内含 `├ └ │ ─` | 树形图 | box-drawing，按等宽对齐 |
| 代码块内含 `→ ↓ ↑ ←` | 流程图 / 关系图 | box-drawing + 箭头 |
| 左侧竖线 + 缩进 | 引用 | `>` |
| 行内字体加粗（视觉差异明显）| 粗体 | `**...**`（视觉不明显时不加）|

|---|---|---|---|---|---|
| Markdown 标题 `#` | ✓ | ✓ | ✓ | ✓ | ✓ |
| Markdown 表格 | ✓ | ✓ | ✓ | ✓ | △ 偶失真 |

## 3 · ASCII art / 树形图对齐规则

### 字符（标准 box-drawing，**禁用** ASCII `|` `+` `-`）

| 用途 | 字符 | Unicode |
|---|---|---|
| 父子连接（竖）| `│` | U+2502 |
| 中间分支 | `├` | U+251C |
| 末尾分支 | `└` | U+2514 |
| 横线 | `─` | U+2500 |
| 拐角（4 种）| `┐ ┘ ┌ └` | U+2510/2518/250C/2514 |
| 三向交点 | `┬ ┴ ├ ┤` | U+252C/2534/251C/2524 |
| 四向交点 | `┼` | U+253C |
| 箭头 | `← → ↑ ↓ ↺ ↻ ↩ ↪` | U+2190~93/21BA/21BB/21A9/21AA |
| 实心下三角 | `▼` | U+25BC |

### 列宽计算（关键）

| 字符类 | 列宽 |
|---|---|
| ASCII（含 `└── dataset_A` 这类）| 1 |
| box-drawing（`│ ├ └ ─`）| 1 |
| 箭头（`→ ←` 等）| 1 |
| 中文 / 中文标点（含 `、，。：；？！…—（）「」`）| **2** |
| 全角空格 ` ` | **2** |
| Emoji | 通常 **2** |

**对齐策略**：

1. 选定基线列 X（树形图竖线 `│` 都落在第 X 列）
2. 每行计算到内容末尾的列数 L
3. padding = `X - 1 - L` 个空格
4. 然后画 `│`

例：让 `│` 在第 60 列，某行内容 L = 45，则 padding = 14 个空格。

### 飞书代码块字体规律

- **macOS**：SF Mono + 苹方 → 中文 ≈ 2 × ASCII（标准）
- **Windows**：Consolas + Microsoft YaHei → 中文 ≈ 2 × ASCII（标准）

按 2:1 算几乎都准。极个别累积误差导致 ±1 列错位，**粘贴后手动加减空格微调**即可。

### 字符速查（直接复制）

```text
├── 中间分支
│   ├── 嵌套（注意是 │ 不是 |）
│   └── 嵌套
└── 末尾分支
    └── 末尾嵌套

→ ← ↑ ↓ ⇒ ⇐ ↔ ↦ ↺ ↻ ↩ ↪
▼ ▲ ◆ ● ▶ ◀ ■ □
「」 『』 【】 《》 〈〉
─ ━ ━━━ ─── ‒ – —
```

## 4 · 表格处理（cell 内换行的坑）

### 飞书表格

`<br>` **不解析**，会显示成字面文本。三种应对：

| 写法 | 适合 | 操作 |
|---|---|---|
| A. 单行 + `；` 分隔 | bullet ≤ 3 条 | cell 内写 `• 第一条；• 第二条` |
| B. 保留 `<br>` + 提示手调 | bullet ≥ 3 条且想保持视觉 bullet | 粘贴后双击 cell → 删 `<br>` → `Shift+Enter` |
| C. bullet 拆到表格外 | 内容很多 | 表格 cell 写"详见下方"，下方用正式列表 |

**默认用 B**（在回复里**必须**写明"粘贴后这个 cell 需要 `Shift+Enter` 手调"）。

### 钉钉表格

行为同飞书——`<br>` 不解析，用 `Shift+Enter` 手调。

### Notion / 语雀 / Confluence

通常 `<br>` 可解析，可保留写法 B 不加提示。

## 5 · AI 执行步骤

1. **读所有截图**，按出现顺序识别结构块（标题/段落/列表/表格/代码块/树形图等）
2. **确认目标平台**：用户未说就默认飞书；如果截图是飞书截图，反向默认 = 钉钉
3. **按目标平台生成**：
   - 标题用 markdown `#`
   - 表格用 markdown table，cell 内换行按"4 · 表格处理"规则处理
   - 代码块用 ` ``` `；树形图用 box-drawing 字符
4. **对齐 ASCII art**：明确给出基线列、每行 padding 数
5. **包装输出**：
   - 可粘贴的代码块
   - "关键还原点"表格说明做了什么
   - "粘贴方式"说明（含平台特定坑）
6. **不擅自加内容**：截图里没有的字、bullet、列，**绝不补**；模糊看不清的字段标 `<?>` 让用户确认

## 6 · 典型坑

### 坑 1：飞书表格 cell `<br>` 不解析（必踩）

见 4 · 表格处理。**每次输出含表格的内容时主动提醒用户**。

### 坑 2：树形图中英文混排错位

字体精度问题。**粘贴后告诉用户：如果第 N 行 `│` 错位 1-2 列，那行加减空格**。

### 坑 3：粘贴时 markdown 未被识别

**症状**：飞书里看到原始 `#` `|` `-` 字符没渲染。

**解决**：
- 飞书：`Cmd+Shift+V` / `Ctrl+Shift+V` 选"作为 Markdown 粘贴"，或斜杠菜单 `/Markdown` 后粘贴
- 钉钉：粘贴时一般自动识别，否则斜杠菜单选"导入 Markdown"

### 坑 4：原图字号太小看不清

**应对**：
- 优先要求用户提供更清晰局部截图
- 退而求其次：把不确定的字段输出为 `<?>` 让用户填
- **绝不**靠"我觉得应该是这个字"猜

### 坑 5：粗体/斜体的过度推断

只在视觉差异明显（明显加粗 / 明显斜体 / 明显变色）时加 `**` `*`，否则保持纯文本。

## 7 · 调用 prompt 模板（在任何 AI 工具里复制即用）

```text
按以下"截图→跨平台文档1:1还原"规则把我贴的截图转成可粘贴内容。

输入：
- 截图：[贴在下面]
- 源平台：[钉钉/飞书/Notion/语雀/Confluence/其它，可写"自动识别"]
- 目标平台：[默认：飞书]

输出要求：
1. 一段可直接粘贴到目标平台的内容（标题/段落用 markdown，表格用 markdown table，代码块用 ```），按截图原顺序
2. 一个"关键还原点"表，列出做了哪些处理
3. 一个"粘贴方式"清单，包含目标平台特有的坑（飞书表格 cell <br> 不解析、cell 内多 bullet 需粘贴后 Shift+Enter 手调等）
4. 如截图含 ASCII art / 树形图：用 box-drawing 字符（│ ├ └ ─ → ▼），按中文=2列 ASCII=1列计算对齐，明确给基线列 X 和每行 padding 数
5. 截图里没有的字、bullet、列绝不补；模糊看不清标 <?>

[贴截图]
```

## 反例 / 不适用

- **不是文档**：截图是 UI 界面、设计稿、数据可视化图——这不是本技能范围，用 OCR + 描述类技能
- **截图太脏**：水印多、字模糊、被遮挡——先让用户重新截
- **跨多语言**：日韩越语等中日韩文混排，列宽规则可能不同，效果存疑
